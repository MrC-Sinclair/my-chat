# Agent 架构远期演进路线

> 本文档从 `AGENTS.md` 的「Agent 架构设计规范」章节中提取，记录尚未在代码中实现的远期规划。当前生效的 Agent 规范见 `AGENTS.md`。

## 长期记忆系统

引入向量库（embedding + 检索）时，检索能力作为**工具暴露给 LLM 自主调用**（Agentic RAG，路径 A），而非每次请求预检索注入（Workflow，路径 B）。确需预检索的场景须在设计文档说明理由。

## 状态管理

- **任务状态独立存储**：长程任务的任务状态（plan/todo/step）独立于 `messages` 表，不混入对话历史
- **工件持久化**：图片、生成的文件、工具输出等单独存储（文件系统/对象存储），消息只存引用（URL/ID）
- **异步持久化**：状态变更异步写库，不阻塞主循环
- **检查点恢复**：长程任务支持从最近检查点恢复，避免崩溃重跑

## 显式任务规划

引入 plan→execute→reflect 显式规划循环后，规划结果作为 reasoning 暴露给前端，不静默执行。

## 自我修复能力

长程 Agent 引入后，允许 LLM 识别失败并主动调整策略（反思→重规划）。
