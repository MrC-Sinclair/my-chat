# 语音输入功能

## 为什么做（Why）

在移动端（手机和平板）上打字效率较低，语音输入可以极大提升用户的输入体验。浏览器原生 `Web Speech API`（`SpeechRecognition`）无需额外依赖，可以直接将语音转为文字填入输入框。

## 改什么（What）

在 `ChatInput.vue` 的发送按钮**左侧**添加一个麦克风按钮，点击后启动语音识别，识别到的文字追加到输入框中。按钮状态随识别过程变化：待机（灰色麦克风图标）、录音中（红色脉冲动画）、浏览器不支持（隐藏/禁用）。

## 不改什么

- 不涉及服务端 API 变更
- 不添加额外的 npm 依赖（使用浏览器原生 API）
- 不改动消息发送/流式输出的逻辑
- 不改变现有键盘输入行为

## 范围

| 层级 | 影响 |
|------|------|
| 前端组件 | `ChatInput.vue`（添加语音按钮和相关逻辑） |
| 页面 | `ai-chat.vue`（接入语音错误提示） |
| 其他 | 无 |

## 成功标准

1. 发送按钮左侧出现麦克风 SVG 图标按钮
2. 不支持 SpeechRecognition 的浏览器中按钮隐藏
3. 点击后开始录音，按钮变为红色脉冲动画状态
4. 再次点击或语音识别结束，识别文字追加到输入框末尾
5. 语音按钮手机端触摸目标 ≥ 44px（核心操作区标准）
6. 纯图标按钮使用 `v-tooltip` 提供文字提示
