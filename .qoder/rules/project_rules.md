# Qoder 环境专属规则

通用规则已写入 `AGENTS.md` 的「AI Agent 执行纪律」章节，本文件仅记录 Qoder 专属补充规则。

## 执行纪律（Qoder 环境）

### 禁止未执行就标记完成

任务标记完成前，必须已经实际执行了对应操作并验证成功。

### 关键操作必须验证

| 操作               | 验证方式                             |
| ------------------ | ------------------------------------ |
| 启动数据库         | `docker compose ps` 确认容器 running |
| 启动开发服务器     | 检查终端输出无 ERROR，或访问页面确认 |
| 安装依赖           | 检查命令退出码为 0                   |
| 数据库 Schema 同步 | 检查命令退出码为 0，无报错           |
| 代码修改           | 运行 `pnpm lint` 和 `pnpm typecheck` |

### 交叉验证

当工具返回的结果会影响后续决策时，必须用另一种工具交叉验证：

- 工具说文件不存在 → 用 `Test-Path` 或 `ls` 再确认
- 命令输出看似成功 → 检查退出码是否为 0
