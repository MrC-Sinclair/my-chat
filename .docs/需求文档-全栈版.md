## 模块一：AI 智能对话 (`/ai-chat`)

### 技术选型核心：Vercel AI SDK

> **为什么用 Vercel AI SDK 而不是手写 SSE？**
>
> | 维度 | 手写 SSE | Vercel AI SDK |
> |------|---------|---------------|
> | 流式处理 | 自己解析 EventSource | `useChat()` 一行搞定 |
> | 多模型切换 | 每家 API 都不同 | 统一接口，换 provider 只改配置 |
> | 错误重试 | 自己实现 | 内置 |
> | 中断生成 | 自己管理 AbortController | `stop()` 方法 |
> | 工具调用(进阶) | 从零实现 | `useChat({tools: ...})` |
> | 代码量 | ~500行 | **~30行** |

### 核心能力

| 能力点 | 描述 | 实现方式 |
|-------|------|---------|
| 多轮对话 | 连续问答，语境连贯 | `useChat()` 自动维护 |
| 流式输出 | 逐字返回，打字机效果 | Vercel AI SDK 内置 Streaming |
| Markdown 渲染 | 公式/表格/代码块 | `ai-sdk/react` 的 `useMarkdownTextStream` |
| 数学公式 | LaTeX 行内/块级 | KaTeX 集成到 Markdown 渲染层 |
| 代码高亮 | 多语言语法高亮 | shadcn-vue 内置或 highlight.js |
| 学情感知 | 结合学生数据理解水平 | System Prompt 注入用户画像 |

import { openai } from '@ai-sdk/openai';
import { streamText } from 'ai';

export default defineEventHandler(async (event) => {
  const { messages, context } = await readBody(event);
  return streamText({
    model: openai('gpt-4o'),
    system: `你是深圳外国语学校的AI学习助手。学生信息：${context.grade}${context.subject}`,
    messages,
    onFinish: async ({ text }) => {

      await saveChatHistory(messages, text);
    },
  }).toDataStreamResponse();
});

## 与前版本文档的差异

| 变更项 | 上一版 (V1.2-Mini) | 本版 (V2.0-FullStack) | 影响 |
|-------|-------------------|---------------------|------|
| **框架** | Vue 3 + Vite (SPA) | **Nuxt 4 (Full-Stack)** | 前后端合一，Server Route 替代独立后端 |
| **AI 方案** | 手写 SSE (`useSSE.ts`) | **Vercel AI SDK (`useChat`)** | 500行→30行，内置流式/多模型/工具调用 |
| **UI 库** | 自定义组件库 | **shadcn-vue + Tailwind** | 开箱即用的美观组件 |
| **状态管理** | Pinia | **Nuxt useState + Pinia** | SSR 友好 |
| **后端 ORM** | 未指定 | **Drizzle ORM + PG18** | 类型安全 SQL，迁移简单 |
| **基础设施** | 未涉及 | **Docker Compose** | 一键启动 PG |
| **路由** | Vue Router | **Nuxt File-based Routing** | 基于文件系统自动生成 |
| **部署** | WebView 嵌入 | 同左 + **可独立部署** | 开发调试更方便 |
